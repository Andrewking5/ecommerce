# 響應式設計檢查清單 (Responsive Design Checklist)

## 📱 目的
此清單用於系統性地檢查所有組件在不同設備尺寸下的顯示效果，確保用戶體驗的一致性。

## 🎯 斷點標準 (Tailwind CSS)
- **Mobile (手機)**: < 640px (sm)
- **Tablet (平板)**: 640px - 1024px (sm - md)
- **Desktop (桌面)**: ≥ 1024px (md+)
- **Wide (寬屏)**: ≥ 1280px (lg+)

## ✅ 通用檢查項目

### 1. 佈局 (Layout)
- [ ] 所有容器在手機版使用單欄佈局
- [ ] 平板版使用 2 欄佈局
- [ ] 桌面版使用多欄佈局
- [ ] 使用 `container-apple` 類別確保適當的 padding
- [ ] 避免水平滾動條出現
- [ ] 內容不會溢出視窗邊界

### 2. 導航 (Navigation)
- [ ] 桌面版顯示完整導航欄
- [ ] 手機版使用漢堡選單 (hamburger menu)
- [ ] 手機版選單包含所有重要連結
- [ ] 語言切換器在手機版菜單中可見
- [ ] 選單可以正常打開和關閉
- [ ] 選單項有適當的觸控目標大小 (≥ 44px)

### 3. 按鈕和互動元素
- [ ] 按鈕在手機版有足夠的觸控區域 (≥ 44x44px)
- [ ] 按鈕文字在手機版不會被截斷
- [ ] 懸停效果在桌面版正常工作
- [ ] 觸控反饋在手機版明顯
- [ ] 按鈕間距適當，不會誤觸

### 4. 文字和排版
- [ ] 文字大小在手機版易讀 (≥ 14px)
- [ ] 標題在手機版適當縮小
- [ ] 行高適當，不會過於緊密
- [ ] 長文字會自動換行
- [ ] 多語言文字不會溢出容器

### 5. 圖片和媒體
- [ ] 圖片響應式縮放 (`w-full`, `h-auto`)
- [ ] 圖片不會變形 (使用 `object-fit`)
- [ ] 圖片載入有適當的佔位符
- [ ] 視頻在手機版可以正常播放

### 6. 表單元素
- [ ] 輸入框在手機版有足夠寬度
- [ ] 標籤和輸入框對齊正確
- [ ] 錯誤訊息在手機版完整顯示
- [ ] 提交按鈕在手機版易於點擊
- [ ] 選擇器 (select) 在手機版正常運作

### 7. 卡片和列表
- [ ] 卡片在手機版使用單欄佈局
- [ ] 卡片間距適當
- [ ] 卡片內容不會溢出
- [ ] 列表項在手機版易於點擊

### 8. 模態框和下拉選單
- [ ] 模態框在手機版佔滿螢幕或適當大小
- [ ] 下拉選單在手機版位置正確
- [ ] 下拉選單不會被截斷
- [ ] 點擊外部區域可以關閉

### 9. 語言切換器 (Language Switcher)
- [ ] 桌面版顯示在導航欄
- [ ] 手機版顯示在移動選單中
- [ ] 語言選項在手機版完整顯示
- [ ] 當前語言有明顯標記
- [ ] 切換語言後 UI 立即更新

### 10. 性能
- [ ] 頁面載入速度 < 2 秒
- [ ] 圖片使用適當的格式和大小
- [ ] 動畫流暢 (60fps)
- [ ] 沒有不必要的重排和重繪

## 🛠️ 測試工具和方法

### 瀏覽器開發工具
1. **Chrome DevTools**
   - 按 `F12` 打開開發工具
   - 點擊設備圖標切換響應式模式
   - 選擇預設設備或自定義尺寸

2. **測試設備尺寸**
   ```
   iPhone SE: 375x667
   iPhone 12/13: 390x844
   iPhone 14 Pro Max: 430x932
   iPad: 768x1024
   iPad Pro: 1024x1366
   Desktop: 1920x1080
   ```

### 手動測試步驟
1. 打開網站
2. 使用開發工具切換到手機模式
3. 檢查所有頁面的佈局
4. 測試所有互動功能
5. 切換語言檢查多語言顯示
6. 測試表單提交
7. 測試購物流程

### 自動化測試 (可選)
- 使用 Playwright 或 Cypress 進行響應式測試
- 截圖對比不同設備尺寸
- 檢查斷點切換是否正常

## 🚀 最佳實踐

1. **Mobile First 設計**
   - 先設計手機版，再擴展到桌面版
   - 使用 `min-width` 媒體查詢

2. **使用 Tailwind 響應式前綴**
   ```tsx
   // 手機版優先，桌面版覆蓋
   <div className="text-sm md:text-base lg:text-lg">
   ```

3. **觸控友好**
   - 按鈕最小 44x44px
   - 適當的間距避免誤觸
   - 使用 `touch-action` 優化觸控

4. **性能優化**
   - 使用 `loading="lazy"` 延遲載入圖片
   - 使用適當的圖片格式 (WebP, AVIF)
   - 避免不必要的重排

5. **無障礙設計**
   - 適當的對比度
   - 鍵盤導航支持
   - 屏幕閱讀器友好
